## 訓練流程

```
1. 資料擷取
   └─ pybaseball：比賽數據、投手數據
   └─ MLB Stats API：即時數據
   └─ 台灣運彩：赔率

2. 特徵工程（feature_builder.py）
   └─ 投手滾動統計（近3/5/10場）
   └─ 牛棚疲勞度計算
   └─ 主客場表現
   └─ 對戰歷史（H2H）

3. 模型訓練（model_trainer.py）
   └─ XGBoost 訓練
   └─ 交叉驗證
   └─ 模型儲存（.pkl + .meta.json）

4. 預測（daily_predictor.py）
   └─ 讀取模型
   └─ 計算去水勝率（Devig）
   └─ 計算期望值（EV）
   └─ 信心指數（55-75）
```

## 預測輸出格式

```
比賽: PIT vs TOR
盤口: 讓分 | 赔率: -105
模型勝率: 56.2%
去水勝率: 54.8%
Edge: 1.4%
EV: 0.66
信心指數: 62
隊伍: 客場 (TOR)
```

## 風險控制

### Kelly Criterion
- 下注比例 = Edge / Odds
- 實際使用 1/4 Kelly（降低波動）
- 只在 Edge > 2% 時建議下注

### 去水機制
- 移除 12% 抽水（vig=0.12）
- 避免被博彩公司稀釋的赔率誤導

### 信心指數
- 範圍：55-75
- 低於 55：不建議下注
- 高於 75：強烈建議

## 模型限制

1. **投手數據依賴**：春訓資料缺少投手統計，模型輸出固定勝率
2. **時區問題**：赔率匹配可能有時間偏移
3. **樣本偏差**：2025年數據可能與歷史分布不同
